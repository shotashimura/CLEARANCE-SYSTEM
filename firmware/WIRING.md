# モーター駆動 配線・設計案（層3〜4）

M5Stack CoreS3 → PWM → ESC / ステアリングサーボ → モーター / 操舵、
電源は 3S LiPo。WPL C74 シャーシに搭載する。**まず1台でベンチテスト**してから量産。

> ⚠️ この段階に進む前に「OSC受信（画面表示）」は動作確認済みであること。
> 本書はその先の「受信値 → 実モーター」の配線設計。

---

## 1. 電源アーキテクチャ（3系統に分ける）

```
3S LiPo (11.1V / XT60)
   │
   ├──[XT60]──▶ ESC ──(大電流)──▶ ブラシモーター 55T   ← 走行(スロットル)
   │
   └──[分岐]──▶ DC-DCコンバータ(12V→5V) ──▶ 5V レール
                                              ├──▶ ステアリングサーボ 20kg (電源+)
                                              └──▶ (任意) M5Stack 5V

M5Stack CoreS3 の電源は次のどれか:
   (推奨・ベンチテスト)  USB-C 給電（PCやモバイルバッテリー）= サーボ電源から分離され最も安全
   (本番案A)            ESC の BEC 5V から（軽負荷・サーボと分離できる）
   (本番案B)            上の DC-DC 5V レールから（要・容量と平滑コンデンサ）
```

**重要な設計判断：M5Stack の電源をサーボと分けるか**
20kg サーボは動作時に大電流（ストールで2〜3A）を引き、同じ5VレールにM5Stackを繋ぐと
**電圧降下でM5Stackがリセット**することがある（ブラウンアウト）。対策：
- ベンチテストは **M5をUSB給電**にして完全分離（最初はこれ一択）
- 本番で1電源化するなら **DC-DCは3A以上（5A推奨）**＋5Vレールに **470〜1000µFの電解コンデンサ**をサーボ近くに追加。可能なら M5 は別系統（ESC BEC か小型バック）に。

## 2. 信号配線（CoreS3 → ESC / サーボ）

CoreS3 の **Grove Port A** から2本の GPIO を PWM 出力に使う（信号は2本だけでよい）。

| CoreS3 ピン | 行き先 | RCコネクタの線 | 役割 |
|---|---|---|---|
| GPIO（ESC_PIN）= Grove G1 | ESC 信号 | 白/オレンジ（信号） | スロットルPWM |
| GPIO（STEER_PIN）= Grove G2 | サーボ 信号 | オレンジ/黄（信号） | 操舵PWM |
| GND | ESC GND **と** サーボ GND | 黒/茶 | **共通グランド（必須）** |

> CoreS3 の Grove Port A は通常 **G1=GPIO1 / G2=GPIO2**（M5の表記）。
> 実機の CoreS3 ピンアウトで GPIO 番号を確認し、`.ino` の `ESC_PIN` / `STEER_PIN` に設定する。
> ESP32-S3 は LEDC でどの GPIO からも 50Hz PWM を出せる。

**配線の注意（ハマりどころ）**
1. **共通グランド必須**：M5Stack の GND と ESC/サーボ の GND を必ず繋ぐ。繋がないとPWMが不安定になる原因No.1。
2. **ESC BEC の +5V を M5 の 5V に挿さない**（M5をUSBやDC-DCで別給電する場合）。信号線とGNDだけ繋ぐ。+を二重給電すると逆流して危険。
3. ESC・サーボの3ピンコネクタ（信号/＋/GND, 2.54mmピッチ）へは Groveケーブル＋ジャンパー（デュポン）で中継。

## 3. PWM パラメータ（RC標準）

```
周波数 : 50Hz（周期 20ms）
パルス幅:
  ESC   1000µs=最大後退 / 1500µs=停止(ニュートラル) / 2000µs=最大前進  ※双方向ブラシESC
  サーボ 1000µs=左いっぱい / 1500µs=中央 / 2000µs=右いっぱい
ESP32 LEDC: 50Hz, 16bit → duty = µs / 20000 * 65535
```

**ESCのアーミング手順**（最初に必ず）
1. 電源投入時、ESCへ **1500µs（ニュートラル）を数秒**送る → ESCが「ピピッ」と鳴って準備完了
2. （ESCによっては初回スロットルキャリブレーション＝full-high/full-lowが必要。マニュアル確認）
3. アーミング前にスロットルを送ると暴走・無反応の原因になる

## 4. OSC値 → 動きへのマッピング

中央サーバーが送る値（`/suitcase/<ID>/*`）を CoreS3 で µs に変換：

```cpp
// スロットル: speed は符号付き（DENIED で負=後退）。安全のため低速から。
float t = constrain(vSpeed, -1.0, 1.0);          // まず [-1,1] にクランプ
float escUs = 1500 + t * SPEED_GAIN;             // SPEED_GAIN=300 から（±300µs=控えめ）

// 操舵
float s = constrain(vSteering, -1.0, 1.0);
float steerUs = 1500 + s * STEER_GAIN;           // STEER_GAIN=300 から（首振り過大を防ぐ）
```

- **最初は GAIN を小さく**（±300µs）して、ゆっくり・小舵から。安定したら広げる。
- `behavior`（hesitant/frozen/random_walk/assertive）は、M5側で
  「進む↔止まる」パターンや揺らぎに反映（次段で実装）。今はまず speed/steering の連続制御から。
- `duration` / `jitter` / `hesitation` は behavior 実装時に使用。

## 5. 1台ベンチテスト手順（安全第一）

```
① タイヤを浮かせる or モーターをドライブシャフトから外す
   → 暴走して机から飛ばないように。これ最重要。
② LiPo を満充電・電圧チェッカーで確認（3S=12.6V付近）。耐火バッグの上で作業。
③ 配線：LiPo→ESC(XT60) / LiPo→DC-DC→サーボ / CoreS3 G1→ESC信号, G2→サーボ信号, GND共通
   M5Stack は USB-C 給電（サーボ電源と分離）
④ .ino の motionSetup() を有効化 → 1500µs を送ってESCアーミング（ピピッ）
⑤ oscSend.js で /suitcase/1/speed を 0 → 小さく → と送り、モーターがゆっくり回るか
⑥ /suitcase/1/steering で前輪が左右に振れるか
⑦ OK なら GAIN を少しずつ上げて調整
```

## 6. 安全・運用メモ

- **LiPo**：過放電（1セル3.0V以下）厳禁。展示中はバッテリーチェッカーで監視、予備と交互運用。充電は耐火バッグ内。
- **逆接続注意**：XT60 の極性、DC-DC の入出力（IN/OUT）を必ず確認してから通電。
- **共通グランド**を最初に確認（不具合の大半はGND未接続）。
- **ESCの発熱**：防水ESCでも長時間は熱を持つ。筐体内の放熱を考慮。
- **電流配線**：モーター・ESC・バッテリー間は太線（AWG12シリコンワイヤー）。信号線は細線でOK。
